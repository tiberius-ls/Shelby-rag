# Shelby RAG

A document Q&A pipeline ‚Äî upload text files, ask questions, get AI-generated answers grounded in the content you uploaded.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` ‚Äî run the API server (port 8080)
- `pnpm --filter @workspace/web run dev` ‚Äî run the React frontend
- `pnpm run typecheck` ‚Äî full typecheck across all packages
- `pnpm run build` ‚Äî typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API hooks and Zod schemas from the OpenAPI spec

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5 + multer (file uploads)
- AI: Anthropic Claude via Replit AI Integrations (`@workspace/integrations-anthropic-ai`)
- Frontend: React + Vite + Tailwind + shadcn/ui + TanStack Query
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` ‚Äî OpenAPI contract (source of truth)
- `artifacts/api-server/src/lib/rag.ts` ‚Äî RAG pipeline: chunker, inverted index, Claude query
- `artifacts/api-server/src/routes/rag.ts` ‚Äî `/rag/documents`, `/rag/upload`, `/rag/query`
- `lib/integrations-anthropic-ai/` ‚Äî pre-configured Anthropic SDK client
- `artifacts/web/src/pages/home.tsx` ‚Äî main UI: upload + query workspace

## Architecture decisions

- **Mock mode only**: document chunks stored in-memory (lost on restart). The Shelby Protocol on-chain storage (`@shelby-protocol/sdk`) is supported by the original code but not wired ‚Äî swap `storeChunk`/`shelbyDownload` in `rag.ts` to go live.
- **Inverted keyword index**: simple token-based search with stop word filtering. No embeddings/vector DB.
- **Claude for generation**: `claude-sonnet-4-6` answers grounded strictly in retrieved context chunks.
- **No DB needed**: the RAG index is all in-memory ‚Äî no Postgres schema required for this pipeline.
- **Anthropic via Replit integrations**: no user API key needed; billed to Replit credits.

## Product

Users upload `.txt` documents, which get chunked and indexed. They then ask natural-language questions and receive precise, source-cited answers generated by Claude from the most relevant chunks.

## User preferences

- Dark, terminal-adjacent UI aesthetic

## Gotchas

- In-memory store: uploaded documents are lost on server restart. For persistence, wire in a real store (Shelby testnet or a DB).
- `SHELBY_MODE=live` + `SHELBY_API_KEY` env var needed to enable on-chain storage via Shelby Protocol.
- Always run `pnpm --filter @workspace/api-spec run codegen` after editing `openapi.yaml`.
